## 贡献类型一览

| 类型 | 方式 | 需要权限 |
|---|---|---|
| 新增主题 | Pull Request | 无，任何人 |
| 更新自己的主题 | Pull Request | 原作者本人即可 |
| 更新他人的主题 | Pull Request | 需在 PR 中 `@原作者` 说明并取得同意；或原作者长期失联时由维护者判断 |
| 反馈主题 Bug | Issue | 无 |
| 主题需求 / 点单 | Issue | 无 |

## 提交一个新主题

### 1. Fork & Clone

```bash
# Fork 本仓库后
git clone https://github.com/<你的用户名>/gridea-pro-themes.git
cd gridea-pro-themes
git checkout -b add-theme-<主题名>
```

### 2. 在 `themes/` 下建一个目录

目录名就是主题名。用 kebab-case（小写 + 连字符），例如 `minimal-dark`、`paper-cards`。

```
themes/
└── your-theme-name/
    ├── config.json                        ← 必需
    ├── README.md                          ← 必需
    ├── templates/                         ← 必需（模板文件）
    ├── assets/
    │   └── media/
    │       └── preview.png                ← 必需，1200×800，< 500KB
    └── LICENSE                            ← 推荐（作者自选，未放默认 MIT）
```

> **预览图路径固定为 `assets/media/preview.<ext>`**，Gridea Pro 应用内「主题选择器」和本仓库画廊都从这里读取。支持 `.png` / `.jpg` / `.jpeg` / `.webp`。

详细字段请看 [docs/theme-spec.md](./docs/theme-spec.md)。

### 3. 最小可用 `config.json`

```json
{
  "name": "your-theme-name",
  "version": "1.0.0",
  "author": "你的名字 / GitHub 用户名",
  "description": "一句话描述你的主题，让人想点进来看看。",
  "engine": "jinja2",
  "templateEngine": "jinja2",
  "customConfig": []
}
```

支持的 `engine` / `templateEngine`：`jinja2`、`ejs`、`go`。详见主题规范。

### 4. 主题自己的 `README.md`

建议包含：

- 一张大的预览图（引用 `preview.png`）
- 主题特色（2-3 句）
- 自定义参数说明（可选）
- 作者署名、授权、联系方式

### 5. 提 PR

PR 标题建议：

```
[new] 新增主题: your-theme-name
```

PR 描述里贴一张预览图更容易被合。

## 更新一个已有主题

1. 只修改你那一个主题目录内的文件
2. **递增 `config.json` 的 `version`**（遵循 semver：bug 修 patch，加功能 minor，破坏性改动 major）
3. 在主题的 `README.md` 或目录里加个 `CHANGELOG.md` 写写改了啥（非必需但受欢迎）
4. 不是你的主题？请先在 PR 里 `@原作者` 征得同意，或在原主题 Issue 下先讨论

PR 标题建议：

```
[update] your-theme-name: v1.0.0 → v1.1.0
```

## 反馈主题问题

所有主题相关反馈请到本仓库提 [Issue](https://github.com/Gridea-Pro/gridea-pro-themes/issues)。我们准备了三个模板：

- **主题 Bug** — 主题显示异常、报错、渲染问题
- **求主题 / 点单** — 希望有某种风格的主题
- **讨论** — 其它一切想聊的事

> Gridea Pro **应用本身**的 bug 请到[主项目仓库](https://github.com/Gridea-Pro/gridea-pro/issues)反馈。

## 关于授权

### 仓库本身

本仓库根目录的 [LICENSE](./LICENSE) 是 **MIT**，**只覆盖仓库基础设施**：README、CONTRIBUTING、docs、CI 脚本、Issue/PR 模板等。

### 主题

每个主题作者**保留自选授权的权利**。推荐在你的主题目录内放一个 `LICENSE` 文件：

```
themes/your-theme-name/LICENSE
```

常见选择：

- **MIT** — 最宽松，别人随便用
- **Apache 2.0** — 宽松 + 专利条款
- **GPL v3** — 衍生作品必须继续开源
- **CC BY 4.0** — 适合偏向"设计资源"的主题
- **自定义** — 允许，但请写清楚

**未声明 LICENSE 时默认视为 MIT。** 提 PR 即视为你同意这条规则。

### 版权声明

提交主题即代表你确认：

- 这是你原创的作品，或你有权将其发布为开源
- 使用的第三方资源（字体、图片、字体图标等）你都确认过授权允许
- 你同意以你声明的 LICENSE（或默认 MIT）公开发布

## 审核流程

我们希望审核是**轻量的**，不想让贡献者等到心累。

| 阶段 | 做什么 | 谁做 |
|---|---|---|
| 1. 自动校验 | GitHub Actions 检查必需文件齐全、`config.json` 合法 | CI |
| 2. 人工初看 | 维护者本地下载、运行、看看效果 | 维护者 |
| 3. 反馈 | 有问题贴评论，有建议也贴评论（非必改） | 维护者 |
| 4. 合并 | 通过后合并到 `main`，画廊自动更新 | 维护者 |

**我们不会因为下列原因拒绝 PR**：

- 代码风格和其它主题不一致
- README 写得不够详细
- 预览图审美你自己觉得一般
- 和已有主题风格相似

**我们可能会要求你修改**：

- 必需文件缺失
- `config.json` 有语法错误或字段缺失
- 疑似抄袭他人主题（未获授权的搬运）
- 包含敏感 / 违规内容

## 命名与规范

- **主题目录名**：`kebab-case`，小写，纯英文/数字/连字符，不超过 30 字符
- **主题 `name` 字段**：展示名，允许空格、中英文，无需与目录名一致
- **唯一性**：目录名不能和已有主题重名
- **预览图**：`assets/media/preview.png`（也接受 `.jpg` / `.jpeg` / `.webp`），1200×800，< 500KB，别放过期截图
- **版本号**：`major.minor.patch`，首次提交从 `1.0.0` 起

## 常见问题

**Q: 可以提交魔改自别人主题的版本吗？**
可以，但请：(1) 在 README 里注明"基于 xxx 主题修改"并附原链接；(2) 确认原主题授权允许派生；(3) 用一个新的目录名，不要覆盖原主题。

**Q: 支持 Hugo / Hexo 主题直接搬过来吗？**
模板语法一般不兼容，需要重写。但设计风格可以借鉴，记得标注灵感来源。

**Q: 我的主题有付费高级版可以吗？**
可以，但**提交到本仓库的版本必须是完整可用的开源版**。你可以在 README 里引导付费高级版，不能藏功能。

**Q: PR 被合进来以后，我还能删除自己的主题吗？**
可以提 Issue 说明原因，我们会帮你下架。但已经发布的版本会保留在 git 历史里（这是 git 的特性，我们改不了）。
